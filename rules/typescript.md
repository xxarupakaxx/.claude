# TypeScript Rules

TypeScript/TSXファイル編集時に適用されるルール。

## コーディング規約

- 早期リターンパターンを使用し、ネストを避ける
- `any`型の使用を避け、適切な型を定義する
- `as`によるキャストより型ガードを優先する
- 関数は80行以内、ファイルは200行以内を目安に分割
- アロー関数を優先（`function`宣言より）

## インポート順序

1. 外部ライブラリ（react, next等）
2. 内部モジュール（@/で始まるパス）
3. 相対インポート（./や../で始まるパス）
4. 型インポート（type のみのインポート）

## 禁止事項

- `console.log`の本番コードへの残存
- `// @ts-ignore`や`// @ts-expect-error`の安易な使用
- 未使用のインポート・変数
