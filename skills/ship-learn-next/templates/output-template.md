# Ship-Learn-Next 出力テンプレート

## Repプラン構造（ステップ5で使用）

各repを以下の構造で作成:

```markdown
## Rep 1: [具体的な目標]

**Ship目標**: [作る/やること]
**成功基準**: [完了をどう判断するか]
**学べること**: [具体的なスキル/洞察]
**必要なリソース**: [最小限 - このrepに必要なものだけ]
**タイムライン**: [具体的な期限]

**アクションステップ**:
1. [具体的なステップ1]
2. [具体的なステップ2]
3. [具体的なステップ3]
...

**シップ後 - 振り返りの質問**:
- 実際に何が起きたか？（具体的に）
- 何がうまくいったか？何がダメだったか？
- 何に驚いたか？
- 1-10でこのrepは何点？
- 次回は何を変えるか？
```

## 将来のRep構造（ステップ6で使用）

```markdown
## Rep 2: [次のレベル]
**基盤**: Rep 1で学んだこと
**新しい挑戦**: 試す/改善する新しいこと1つ
**予想難易度**: [簡単/同じ/難しい - 理由も]

## Rep 3: [進行を継続]
...
```

**進行の原則**:
- 各repで新しい要素を1つ追加
- 成功に基づいて難易度を上げる
- コンテンツの具体的な教訓を参照
- repをシップ可能に保つ（理論的ではなく）

## プランの保存

**重要**: 必ずプランをファイルに保存する。

### ファイル名規則

常に以下の形式を使用:
- `Ship-Learn-Next Plan - [簡潔なクエストタイトル].md`

例:
- `Ship-Learn-Next Plan - Build in Proven Markets.md`
- `Ship-Learn-Next Plan - Learn React.md`
- `Ship-Learn-Next Plan - Cold Email Outreach.md`

**クエストタイトルの条件**:
- 簡潔（3-6語）
- 主な目標を説明
- コンテンツの核心的な教訓/テーマに基づく

### 保存する内容

**以下を含む完全なプラン**:
- 目標とソースを含むクエスト概要
- すべてのrep（1-5）の完全な詳細
- アクションステップと振り返りの質問
- タイムラインのコミットメント
- ソース素材への参照

**形式**: 可読性のため常にMarkdown（`.md`）で保存
