# Ship-Learn-Next アクションプランナー

受動的な学習コンテンツを実行可能な**Ship-Learn-Nextサイクル**に変換する。

## このスキルを使うタイミング

ユーザーが以下の場合にアクティベート:
- トランスクリプト/記事/チュートリアルがあり「アドバイスを実践したい」
- 「これをプランにして」「実行可能にして」と依頼
- 教育コンテンツから実装ステップを抽出したい
- 大きなアイデアを小さくシップ可能なrepに分解したい
- 「Xを見た/読んだけど、次は何をすべき？」と質問

## コアフレームワーク: Ship-Learn-Next

すべての学習クエストは3つのフェーズを繰り返す:

1. **SHIP** - 実物を作る（コード、コンテンツ、プロダクト、デモ）
2. **LEARN** - 起きたことを正直に振り返る
3. **NEXT** - 学びに基づいて次のイテレーションを計画

**核心原則**: 100時間の学習より100回のrepが勝る。学習 = より多く知ることではなく、より上手くやること。

## ステップ

### ステップ1: コンテンツを読む

ユーザーが提供するファイル（トランスクリプト、記事、メモ）をReadツールで分析する。

### ステップ2: 核心的な教訓を抽出

コンテンツから以下を特定:
- **主なアドバイス/教訓**: 重要なポイントは何か？
- **実行可能な原則**: 実際に練習できることは何か？
- **教えられているスキル**: これを実践すると何を学べるか？
- **例/ケーススタディ**: 言及されている実際の実装

**やってはいけないこと**:
- すべてを要約する（実行可能な部分に集中）
- 応用なしに理論を列挙する
- 「知っておくといい」vs「練習が必要」を混同する

### ステップ3: クエストを定義

質問:
1. 「このコンテンツに基づいて、4-8週間で何を達成したいですか？」
2. 「成功とはどんな状態ですか？（具体的に）」
3. 「何か具体的に作れる/創れる/シップできるものは？」

**良いクエストの例**: 「コールドアウトリーチを10件送り、2件の返信を得る」
**悪いクエストの例**: 「営業について学ぶ」（曖昧すぎる）

### ステップ4: Rep 1を設計（最初のイテレーション）

クエストを**最小のシップ可能バージョン**に分解:
- 「今週シップできる最小バージョンは何ですか？」
- 「それをやるためだけに何を学ぶ必要がありますか？」
- 「rep 1の『完了』とはどんな状態ですか？」

**条件**: 具体的で明確 / 1-7日で完了可能 / 実際の証拠・成果物を生成 / 怖気づかない程度に小さい / 意味のある学びがある程度に大きい

### ステップ5: Repプランを作成

`Read templates/output-template.md` のRepプラン構造テンプレートを使用して各repを作成する。

### ステップ6: 将来のRep（2-5）をマッピング

`Read templates/output-template.md` の将来Rep構造と進行の原則を使用する。

### ステップ7: コンテンツと接続

各repでソース素材を参照する。「これは[概念]を実装」「[テクニック]を練習」のように接続。
**ただし**: 常に学習よりも実行を強調。リソースは特定のrepに必要な場合のみ指し示す。
