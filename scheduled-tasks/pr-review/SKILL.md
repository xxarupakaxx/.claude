---
name: pr-review
description: PR関連の対応
---

Monitor pull requests in ydb-superapp-server.

1. When a comment is posted on a PR you opened: Run pr-review on the PR, summarize findings, and push fixes if needed.
2. When a comment is posted on a PR where you are a reviewer: Run pr-review on the PR and share the results in Slack.
3. For each PR, include the pr-review output, key findings, and next steps.

If pr-review finds no issues, note that briefly. Always notify via Slack when sharing review results.
